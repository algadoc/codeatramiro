¿Como consigue la página leer los archivos con los materiales, decidir cual tiene que enseñar, y mostrar el seleccionado por el usuario?

Buena pregunta. Ni siquiera yo (Alfonso Lagares), el autor original de la página, lo comprendo completamente. Sin embargo, podemos comprender lo suficiente el sistema como para añadir nuevos conenidos. Como conigue la página todo lo demas solo le importa al pobre infeliz de mantenimiento.

Para empezar, comprendamos la estructura de los archivos que llegan al cliente una vez entra en la página. Estos son los presentes en la carpeta /public. Dentro, a su vez, no encontramos con este panorama:

[alagares@localhost public]$ ls
contents    manifest.json  robots.txt
index.html  pageicon.png

Podemos ignorar por completo robots.txt. Eso nos dejas con los siguientes archivos:

  • index.html: Archivo HTML principal de la página. Verás que esta prácticamente vacio. Esto es porque React lo genera dinámicamente. Se recomienda no tocar.
  • manifest.json: Archivo JSON requerido por algunos buscadores, el cual contiene información acerca de como cargar la página. Mejor no tocar.
  • pageicon.png: Icono que aparece en la barra del buscador cuando carga la página. Se puede cambiar por otro png.
  • contents: Esta es la carpeta que nos interesa. Guarda todos los archivos de materiales, y es primordial para el funcionamiento de la página.
[alagares@localhost public]$ cd contents/
[alagares@localhost contents]$ ls
C                   CSS                  HTML        prism.css  Template.html
contentstyling.css  currentcourses.json  Javascript  prism.js
CS101               Devs                 LaTex       Python

Madre mia, cuanta cosa. En realidad, es muy sencillo. Tenemos los siguientes archivos:

  • prism.css: Archivo CSS asociado con la librería de Prismjs, la cual se usa para poder incluir código en los materiales. Se explica un poco mas tarde su uso con detalle, pero por ahora es primordial saber que tiene que ser importada en todos los archivos de materiales que se suban a la página, se use o no.
  • prism.js: Archivo JS asociado con la librería de Prismjs. Es lo que hace que el código aparezca así de bonito. Está fuera de la comprensión de cualquier mortal, por lo que se desaconseja modificarla. Como su archivo CSS correspondiente, se tiene que importar en todos los materiales.
  • Template.html: Archivo HTML que sirve de base para cualquier material que se suba a la plataforma. Contiene numerosos ejemplos de como generar distintos elementos para la creación de materiales.
  • contentstyling.css: Hoja de estilos común para todos los materiales. Tiene que importarse al principio de cada archivo de materiales y unfica el look general de los contenidos. Se puede sobrescribir incluyendo estilos propios dentro del archivo HTML del material, pero eso puede llevar a problemas de estabilidad.
