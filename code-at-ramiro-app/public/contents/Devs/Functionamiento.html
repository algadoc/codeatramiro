Como ya se ha comentado, esta página web funciona con React. Como Developer, no se te pide que mantengas directamente la página, sino que crees contenidos para ella. Esto implica que no tienes por qué conocer su funcionamiento completo. Por lo tanto, lo mas recomendable es que el código fuente en sí no se modifique directamente. Esto no significa que no estes invitado a crear cosas nuevas para la página, pero es primordial que exista una versión estable y funcional, y que ese codigo no se pierda.

Esto demuestra la importancia de el repositorio de Github y el sistema de versionado de Google Firebase. Si eres el responsable de configurar estos servicios, asegúrate de que siempre exista una versión funcional a la que hacer un rollback antes de tocar nada, y que sabes hacerlo. Esto se puede hacer con un servidor de desarrollo, tan facil como seguir las instrucciones en el README.md de dentro de la carpeta de la app de React.

Si no eres el responsable de mantener la página web, tu cometido es crear mas materiales para ampliar la oferta de la página. Por lo tanto, tienes que comprender como la página guarda, presenta y organiza esos materiales.
