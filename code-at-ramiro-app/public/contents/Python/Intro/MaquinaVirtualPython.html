La máquina virtual de Python

Es mas sencillo de lo que parece

¿Máquina virtual? ¿Qué es eso? Suena complicado, pero en realidad es muy sencillo. Máquina, en este caso, se refiere a un ordenador. El ordenador con el que estas viendo esta página web es un objeto físico que utiliza electricidad para realizar dos funciones principales: Almacenar información y realizar operaciones sobre esa información. A ese almacenaje lo llamamos memoria.

Tu ordenador ahora mismo está realizando miles de operaciones, desde calcular el color de los píxeles que tiene que mandar a la pantalla hasta comprobar si te ha llegado un correo nuevo. Todas estas operaciones tienen como objetivo cambiar la información que hay almacenada de alguna manera, con algún objetivo.

Una máquina virtual es, por así decirlo, un ordenador dentro de un ordenador. Es decir, una máquina virtual toma parte de la memoria de tu ordenador y parte de su potencia computacional para que funcione como un ordenador, esta vez sin piezas físicas. Esta máquina virtual acepta instrucciones, las cuales podemos escribir y guardar en la memoria de la máquina para que las ejecute.

La máquina virtual de Python es un programa que se instala en tu ordenador y utiliza sus recursos para ejecutar instrucciones escritas en Python, lo que llamaremos a partir de ahora scripts. A través de estas instrucciones, podemos alterar la información que esta máquina virtual contiene.

Te estarán asaltando un millón de preguntas ahora mismo. ¿Como puede la máquina virtual entender Python, pero mi ordenador no? ¿Como funciona la máquina virtual por dentro? ¿Como interactuan la máquina virtual y mi ordenador? Es bueno cuestionarlo todo, pero esto me ayuda a explicar un concepto crítico en programación, llamado abstracción.

La abstracción consiste en aislar partes de un programa o proceso del resto de partes. Es decir, cuando vemos una parte de un proceso no hay que preguntarse "¿como lo hace?" sino "¿qué hace?". Por ejemplo, volviendo al tema de la cocina: cunado estas friendo unas patatas, ¿necesitas saber como funciona la freidora? ¡No! solo necesitas saber como funciona. En la freidora, metes patatas crudas y salen, después de un rato, patatas fritas.

En programación es exactamente igual. Cuando tenemos un problema muy complejo que resolver,tenemos que dividirlo en problemas mas pequeños, y una vez los solucionamos (o mas a menudo, alguien los resuelve por nosotros) tenemos que olvidarnos de como funciona y centrarnos en qué hace. Volvamos a la cocina para comprender a lo que me refiero.

¿Como hace uno una pizza? Bueno, hacer una pizza es en realidad hacer una masa y una salsa de tomate. De un problema complicado, hemos creado dos problemas mas pequeños y factibles. Entonces, vamos a ver como hacer una pizza:


Hacer masa de pizza
Hacer salsa de tomate
Estirar la masa de pizza
Poner la salsa sobre la masa de pizza
Poner toppings sobre la salsa
Hornear

Esta receta parece correcta, pero si nos fijamos no es completa. ¿Por qué? Bueno, ¿como se hace una masa para pizza, o una salsa de tomate? Para que un ordenador siga unas instrucciones, necesitamos detallarlo todo. Por lo tanto, podemos definir como hacer la masa y la salsa de tomate.


Hacer masa de pizza = {
    Mezclar agua, harina, huevo y levadura
    Amasar hasta que la mezcla sea homogénea
    Dejar reposar cubierta con aceite  
}
    

Hacer salsa de tomate = {
    Picar tomates maduros
    Mezclar los tomates picados, especias y aceite en una olla
    Cocer y reducir hasta que sea los suficientemente densa
}

Si te das cuenta, la receta inicial de la pizza puede ser completada con estas dos piezas mas pequeñas. Podríamos completarla, pero normalmente no hace falta. ¿Por qué? Bueno, normalmente uno compra la masa de pizza y la salsa de tomate ya hecha. Para hacer una pizza, ¿de verdad necesitas conocer como se hace la masa o la salsa, si ya viene hecha?. Eso es abstracción. Dividir el problema en problemas mas pequeños, encontrar soluciones para cada uno de esos pequeños problemas y una vez solucionados olvidarse de como funcionan esas soluciones y centrarse en qué hacen.

Así que, vamos a recapitular:

  • Python funciona dentro de una máquina virtual, la cual almacena información y realiza operaciones sobre esa información.
  • Un script te permite interactuar con la máquina virtual a través de unas instrucciones.
  • La abstracción es primordial para crear programas, y te permite solucionar problemas muy complicados. Consiste en:
    • Dividir el problema principal en otros problemas mas pequeños.
    • Solucionar cada pequeño problema.
    • Olvidarse de como funciona cada pequeño programa y centrarse en qué hacen.
