Comentarios

O como programar sin programar

Programar mola, pero muchas veces tenemos que explicar que narices estamos haciendo en nuestro código a otros programadores. Esto se hace con comentarios. Comentar tu código es una práctica muy recomendada, ya que hace arreglar problemas mucho mas facil y te mantiene ordenado.

¿Y como hacemos que Python no ejecute un trozo de texto? Tenemos dos formas. Una, si queremos hacer un comentario de una sola linea, es añadir el símbolo # al principio de la linea.

#Esto es un comentario, y no se ejecutará. El comando de debajo si se ejecutará
print()

La otra manera es utilizar las triples dobles comillas, """, y permite hacer comentarios de múltiples lineas. Esto significa que cuando se comienza un comentario hay que cerrarlo con otro para seguir programando.

"""Esto es un comentario multilinea,
es decir, que esto también estará comentado. Cuando lo cierre, dejará de ser un comentario"""
print()

Se utiliza más la almohadilla, ya que se busca que los comentarios sean breves y concisos.
